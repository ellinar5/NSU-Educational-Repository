ВАЖНО: 
1)все функции должны принимать аргумент;
2)все функции должны возвращать значение;
3)функция возвращает одно и то же значение для одного аргумента;

Команды в строке Prelude:
1):q выйти из GHCi
2):? список доступных команд и их краткое описание 
3):l <имя программы> загрузка программы с  указанным именем
4):r перегрузка текущего модуля
5):t <имя функции> печать типа выражения 
6):i <имя функции> печать сигнатуры функции

Функции:
1)div - целочисленное деление
2)mod - модуль числа
3)min - нахождение минимального числа
4)max - нахождение максимального числа
5)sqrt - квадратный корень
6)succ - возвращает следущее целое число (последовательность)
7)quot - целочисленное деление,  НО может вести себя по-другому в крайних случаях (например, отрицательные числа).
8)log - натуральный логарифм (по основанию е)
9)logBase - логариф у которого можно выставлить основание

2)exp2 :: Int -> Int -> Int -> Int 
exp2 b с d = b * c * d 

Переменные:
Замыкания (локальные определения)

Дальними аналогами локальных присваиваний в императивных языках в функциональном языке Haskell являются два способа порождать локальные определения.
● let-выражения
● where-конструкция


Условные инструкции (операторы)
Prelude> let f x = if x > 0 then 1 else (-1)
Prelude> f 7
1

Блок where
calcChange owed given = if change > 0
			then change
			else 0
where change = given - owed

given - owed вычисляется единожды и присваивается переменной change.

Охранные выражения
example n | n > 1 = "Good"
	  | n > 0 = ">0"
	  | otherwise = "Error!"

Лямбда
Наиболее важных идей функционального программирования — функция без имени,
называемая лямбда-функцией (отсюда происходит название лямбда-исчисление).
Лямбда-функции часто обозначаются с помощью маленькой греческой буквы λ.
Другое распространённое название лямбда-функции — анонимная функция.

\x -> x

GHCi> (\x -> x) 4
4
GHCi> (\x -> x) "NSU"
NSU
GHCi> (\x -> x) [1,2,3]
[1,2,3]

Обратите внимание: каждый раз при использовании лямбда-функции
нужно писать её определение

Компазиция
Композиция означает, что вы можете создать что-то новое, скомбинировав
две похожие сущности.

До сих пор для комбинирования нескольких строк вы использовали
операцию ++. Это может быть утомительно для больших строк:
"это" ++ " " ++ "немного" ++ " " ++ "слишком"

Специальная функция высшего порядка под названием композиция,
записывающаяся как (.) — точка, принимает две функции в качестве
аргументов.

Использование композиции для функций очень удобно для
комбинирования функций на лету читаемым образом.
Ниже приведены некоторые примеры функций, которые могут быть
легко выражены с помощью композиции функций.


Операторы:
. - можно не ставить скобки
$ - заменяет перднию скобку
